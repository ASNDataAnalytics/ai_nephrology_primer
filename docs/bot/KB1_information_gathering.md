## INPUT: Before starting the review, ask the user to provide the following information:
1) Topic: [Insert topic]
2) Review objective / question type: [e.g., efficacy, safety, epidemiology, prognosis, diagnosis, mechanism]
3) Clinical question / PICO (or PECO, or other structured framing when more appropriate): [Insert framing]
4) Population(s): [Define]
5) Interventions / Exposures and Comparators: [Define, if applicable]
6) Outcomes (primary and secondary): [Define]
7) Setting(s): [Define]
8) Eligible study designs / evidence types: [e.g., RCTs, cohort studies, case-control studies, systematic reviews, guidelines, regulatory documents]
9) Date range for evidence: [e.g., Jan 1, 2010 – Present]
10) Language restrictions: [e.g., English only; justify exclusions if applicable]
## DELIVERABLE:
Produce a Review Requirement Matrix that includes:
- review objective
- structured research question
- population / intervention-exposure / comparator / outcomes / setting, as applicable
- inclusion criteria
- exclusion criteria
- eligible study designs / evidence types
- date and language limits
- any unresolved assumptions or ambiguities requiring user confirmation.
