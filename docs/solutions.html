Response:

Below is a secure project scaffold and Python implementation pattern for identifying potentially eligible patients for this trial from an EPIC Clarity environment.

Two important realities up front:

  1. Some trial criteria are often not available as reliable structured Clarity fields and therefore cannot be used for a fully automated eligibility determination without local enrichment, manual review, or both.

  2. Exact Clarity table/column names and category values vary by organization and Epic version, so the SQL extraction layer should be treated as a local mapping step. The Python below is valid and production-oriented; the SQL includes clearly marked places where your local Clarity analyst should map the exact source tables/columns.

Polars can read query results from a database using pl.read_database / pl.read_database_uri, SQLAlchemy supports database URLs and connection configuration, and Microsoft’s ODBC Driver 18 enables encrypted SQL Server connections by default.

What can be automated vs. what cannot

Usually automatable or semi-automatable from Clarity

These are often available in structured form somewhere in Clarity or derivable from it:

  • race / ethnicity proxy for “self-identifies as Black or African American”

  • age

  • CKD stage 1–4, if derived from diagnoses and/or eGFR labs

  • preferred language = English

  • active phone on file

  • diagnoses suggesting kidney transplant

  • diagnoses suggesting severe cognitive impairment, unstable CAD/angina, orthostatic hypotension, lower-extremity amputation

  • durable medical equipment / flowsheet / problem-list proxies for wheelchair / walker / cane dependence

Usually not fully automatable from Clarity alone

These criteria commonly require survey data, research registry data, chart review, or investigator adjudication:

  • self-reported 6+ hours/day sedentary time

  • ability to speak/read/understand English beyond preferred language proxy

  • unable to walk 1 block

  • unable to stand from seated position without assistance

  • using wheelchair/walker/cane for all ambulation

  • orthopedic / neurologic / psychiatric condition that would preclude prompted sit-to-stand transitions

  • any other condition investigator considers precludes participation

  • participated in the Patient Advisory Board unless your org tracks this in a research registry or custom table

So the correct deliverable is a secure pre-screening pipeline that returns:

  • likely_eligible

  • manual_review_required

  • not_eligible

from __future__ import annotations

from pathlib import Path
from urllib.parse import quote_plus

import polars as pl
from sqlalchemy import create_engine
from sqlalchemy.engine import Engine

from trial_eligibility.settings import settings


def build_connection_string() -> str:
    driver = quote_plus(settings.clarity_odbc_driver)
    password = quote_plus(settings.clarity_password)

    return (
        f"mssql+pyodbc://{settings.clarity_username}:{password}"
        f"@{settings.clarity_host}:{settings.clarity_port}/{settings.clarity_database}"
        f"?driver={driver}"
        f"&Encrypt={settings.clarity_encrypt}"
        f"&TrustServerCertificate={settings.clarity_trust_server_certificate}"
    )


def get_engine() -> Engine:
    return create_engine(
        build_connection_string(),
        pool_pre_ping=True,
        future=True,
    )


def read_sql_file(path: str | Path) -> str:
    return Path(path).read_text(encoding="utf-8")


def read_polars(query: str) -> pl.DataFrame:
    engine = get_engine()
    with engine.connect() as conn:
        return pl.read_database(query=query, connection=conn)

7) sql/candidate_extract.sql

This query is intentionally PHI-minimizing. It does not select patient name, address, email, or MRN for output. It uses an internal durable patient key only, which will later be hashed into a study ID.

You must map the placeholder table/column names to your local Clarity schema. The point is to extract only the minimum structured data needed for pre-screening.

/*

- Do not add direct identifiers like patient name, address, or MRN to the final output.

Expected output columns:

  unable_stand_without_assist_flag
  sedentary_hours_per_day
  patient_advisory_board_flag

WITH
-- 1) Core patient demographics
patient_core AS (
    SELECT
        p.PAT_ID                AS patient_id,
        p.BIRTH_DATE            AS birth_date,
        p.PREFERRED_LANGUAGE_C  AS language_code,
        CASE
            WHEN p.HOME_PHONE IS NOT NULL
              OR p.MOBILE_PHONE IS NOT NULL
              OR p.WORK_PHONE IS NOT NULL
            THEN 1 ELSE 0
        END AS has_phone
    FROM CLARITY_PATIENT p
    WHERE p.IS_TEST_PAT_YN = 'N'
),

-- 2) Race
patient_race AS (
    SELECT
        pr.PAT_ID AS patient_id,
        MAX(CASE
            WHEN zr.NAME IN ('Black or African American', 'Black', 'African American')
            THEN 1 ELSE 0
        END) AS black_or_african_american_flag
    FROM PATIENT_RACE pr
    JOIN ZC_PATIENT_RACE zr
      ON pr.PATIENT_RACE_C = zr.PATIENT_RACE_C
    GROUP BY pr.PAT_ID
),

-- 3) Preferred language
patient_language AS (
    SELECT
        zpl.LANGUAGE_C,
        zpl.NAME AS language_name
    FROM ZC_LANGUAGE zpl
),

-- 4) CKD diagnosis proxy
ckd_dx AS (
    SELECT
        dx.PAT_ID AS patient_id,
        MAX(CASE
            WHEN dx.ICD10_CODE LIKE 'N18.1%' THEN 1 ELSE 0
        END) AS ckd_stage_1_dx,
        MAX(CASE
            WHEN dx.ICD10_CODE LIKE 'N18.2%' THEN 1 ELSE 0
        END) AS ckd_stage_2_dx,
        MAX(CASE
            WHEN dx.ICD10_CODE LIKE 'N18.3%' THEN 1 ELSE 0
        END) AS ckd_stage_3_dx,
        MAX(CASE
            WHEN dx.ICD10_CODE LIKE 'N18.4%' THEN 1 ELSE 0
        END) AS ckd_stage_4_dx
    FROM PROBLEM_LIST_DX dx
    GROUP BY dx.PAT_ID
),

-- 5) Latest eGFR lab proxy
latest_egfr AS (
    SELECT patient_id, result_numeric AS latest_egfr
    FROM (
        SELECT
            lr.PAT_ID AS patient_id,
            TRY_CAST(lr.RESULT_VALUE AS FLOAT) AS result_numeric,
            ROW_NUMBER() OVER (
                PARTITION BY lr.PAT_ID
                ORDER BY lr.RESULT_DATE DESC
            ) AS rn
        FROM LAB_RESULT lr
        WHERE lr.COMPONENT_NAME IN ('eGFR', 'Estimated GFR')
          AND TRY_CAST(lr.RESULT_VALUE AS FLOAT) IS NOT NULL
    ) x
    WHERE rn = 1
),

-- 6) Exclusion diagnoses / conditions
exclusion_dx AS (
    SELECT
        dx.PAT_ID AS patient_id,
        MAX(CASE
            WHEN dx.ICD10_CODE LIKE 'Z94.0%' OR dx.ICD10_CODE LIKE 'T86.1%'
            THEN 1 ELSE 0
        END) AS kidney_transplant_flag,

        MAX(CASE
            WHEN dx.ICD10_CODE IN ('I20.0','I20.8','I20.9','I25.11','I25.118','I25.119')
            THEN 1 ELSE 0
        END) AS unstable_cad_flag,

        MAX(CASE
            WHEN dx.ICD10_CODE LIKE 'I95.1%'
            THEN 1 ELSE 0
        END) AS orthostatic_hypotension_flag,

        MAX(CASE
            WHEN dx.ICD10_CODE LIKE 'Z89.5%' OR dx.ICD10_CODE LIKE 'Z89.6%'
            THEN 1 ELSE 0
        END) AS amputation_flag,

        MAX(CASE
            WHEN dx.ICD10_CODE LIKE 'F01%'
              OR dx.ICD10_CODE LIKE 'F02%'
              OR dx.ICD10_CODE LIKE 'F03%'
              OR dx.ICD10_CODE LIKE 'G30%'
            THEN 1 ELSE 0
        END) AS severe_cognitive_impairment_flag,

        MAX(CASE
            WHEN dx.ICD10_CODE LIKE 'F20%'
              OR dx.ICD10_CODE LIKE 'F31%'
              OR dx.ICD10_CODE LIKE 'G20%'
              OR dx.ICD10_CODE LIKE 'G35%'
              OR dx.ICD10_CODE LIKE 'G80%'
            THEN 1 ELSE 0
        END) AS severe_psych_neuro_flag,

        MAX(CASE
            WHEN dx.ICD10_CODE LIKE 'M16%'
              OR dx.ICD10_CODE LIKE 'M17%'
              OR dx.ICD10_CODE LIKE 'M48%'
            THEN 1 ELSE 0
        END) AS ortho_condition_flag
    FROM PROBLEM_LIST_DX dx
    GROUP BY dx.PAT_ID
),

-- 7) Mobility / functional status placeholders
mobility_status AS (
    SELECT
        m.PAT_ID AS patient_id,
        MAX(CASE WHEN m.ALL_AMBULATION_WITH_DEVICE_YN = 1 THEN 1 ELSE 0 END) AS mobility_device_dependence_flag,
        MAX(CASE WHEN m.UNABLE_WALK_ONE_BLOCK_YN = 1 THEN 1 ELSE 0 END) AS unable_walk_block_flag,
        MAX(CASE WHEN m.UNABLE_STAND_FROM_SEATED_WITHOUT_ASSIST_YN = 1 THEN 1 ELSE 0 END) AS unable_stand_without_assist_flag
    FROM PATIENT_MOBILITY_ASSESSMENT m
    GROUP BY m.PAT_ID
),

-- 8) Sedentary behavior placeholder
sedentary_behavior AS (
    SELECT
        s.PAT_ID AS patient_id,
        MAX(s.SEDENTARY_HOURS_PER_DAY) AS sedentary_hours_per_day
    FROM RESEARCH_SCREENING_QUESTIONNAIRE s
    GROUP BY s.PAT_ID
),

-- 9) Research / advisory board participation placeholder
advisory_board AS (
    SELECT
        a.PAT_ID AS patient_id,
        1 AS patient_advisory_board_flag
    FROM RESEARCH_PATIENT_ADVISORY_BOARD a
)

SELECT TOP (@ROW_LIMIT)
    pc.patient_id,
    pc.birth_date,
    COALESCE(pr.black_or_african_american_flag, 0) AS black_or_african_american_flag,
    pl.language_name AS language,
    pc.has_phone,

    CASE
        WHEN COALESCE(cd.ckd_stage_4_dx, 0) = 1 THEN 4
        WHEN COALESCE(cd.ckd_stage_3_dx, 0) = 1 THEN 3
        WHEN COALESCE(cd.ckd_stage_2_dx, 0) = 1 THEN 2
        WHEN COALESCE(cd.ckd_stage_1_dx, 0) = 1 THEN 1
        ELSE NULL
    END AS ckd_stage_dx,

    le.latest_egfr,

    COALESCE(ed.kidney_transplant_flag, 0) AS kidney_transplant_flag,
    COALESCE(ed.severe_cognitive_impairment_flag, 0) AS severe_cognitive_impairment_flag,
    COALESCE(ed.unstable_cad_flag, 0) AS unstable_cad_flag,
    COALESCE(ed.orthostatic_hypotension_flag, 0) AS orthostatic_hypotension_flag,
    COALESCE(ed.amputation_flag, 0) AS amputation_flag,
    COALESCE(ed.severe_psych_neuro_flag, 0) AS severe_psych_neuro_flag,
    COALESCE(ed.ortho_condition_flag, 0) AS ortho_condition_flag,

    ms.mobility_device_dependence_flag,
    ms.unable_walk_block_flag,
    ms.unable_stand_without_assist_flag,

    sb.sedentary_hours_per_day,
    COALESCE(ab.patient_advisory_board_flag, 0) AS patient_advisory_board_flag

FROM patient_core pc
LEFT JOIN patient_race pr
    ON pc.patient_id = pr.patient_id
LEFT JOIN patient_language pl
    ON pc.language_code = pl.LANGUAGE_C
LEFT JOIN ckd_dx cd
    ON pc.patient_id = cd.patient_id
LEFT JOIN latest_egfr le
    ON pc.patient_id = le.patient_id
LEFT JOIN exclusion_dx ed
    ON pc.patient_id = ed.patient_id
LEFT JOIN mobility_status ms
    ON pc.patient_id = ms.patient_id
LEFT JOIN sedentary_behavior sb
    ON pc.patient_id = sb.patient_id
LEFT JOIN advisory_board ab
    ON pc.patient_id = ab.patient_id;

8) src/trial_eligibility/logic.py

This is where the inclusion/exclusion logic is applied with Polars. It intentionally creates a manual-review path whenever required criteria are not available or are only weakly proxied.

Code 
from __future__ import annotations

import hashlib
import hmac
from datetime import date

import polars as pl

from trial_eligibility.settings import settings


def stable_study_id(patient_id: str | int) -> str:
    """
    Irreversible HMAC-based study ID.
    Do not expose raw patient IDs in exported files.
    """
    msg = str(patient_id).encode("utf-8")
    key = settings.study_id_salt.encode("utf-8")
    digest = hmac.new(key, msg, hashlib.sha256).hexdigest()
    return digest[:16]


def infer_ckd_stage_from_egfr(latest_egfr: float | None) -> int | None:
    """
    Conservative eGFR-only approximation.
    Trial includes CKD Stage 1-4 and excludes stage 5.
    Stage 1 usually requires evidence of kidney damage and cannot be inferred from eGFR alone.
    """
    if latest_egfr is None:
        return None
    if latest_egfr >= 90:
        return 1
    if 60 <= latest_egfr < 90:
        return 2
    if 30 <= latest_egfr < 60:
        return 3
    if 15 <= latest_egfr < 30:
        return 4
    return 5


def classify_candidates(df: pl.DataFrame) -> pl.DataFrame:
    today = date.today()

    working = (
        df
        .with_columns([
            pl.col("birth_date").cast(pl.Date),
            (
                pl.lit(today).cast(pl.Date).dt.year()
                - pl.col("birth_date").dt.year()
                - (
                    (
                        (pl.lit(today.month) < pl.col("birth_date").dt.month())
                        | (
                            (pl.lit(today.month) == pl.col("birth_date").dt.month())
                            & (pl.lit(today.day) < pl.col("birth_date").dt.day())
                        )
                    ).cast(pl.Int8)
                )
            ).alias("age_years"),
        ])
        .with_columns([
            pl.col("latest_egfr")
              .map_elements(infer_ckd_stage_from_egfr, return_dtype=pl.Int64)
              .alias("ckd_stage_egfr"),
            pl.coalesce(
                [pl.col("ckd_stage_dx"), pl.col("ckd_stage_egfr")]
            ).alias("ckd_stage_final"),
            pl.col("language").fill_null("UNKNOWN").str.to_uppercase().alias("language_norm"),
            pl.col("sedentary_hours_per_day").cast(pl.Float64, strict=False),
        ])
        .with_columns([
            (pl.col("black_or_african_american_flag") == 1).alias("meets_race"),
            (pl.col("age_years").is_between(35, 80, closed="both")).alias("meets_age"),
            (pl.col("ckd_stage_final").is_between(1, 4, closed="both")).alias("meets_ckd"),
            (pl.col("language_norm") == "ENGLISH").alias("meets_language_proxy"),
            (pl.col("has_phone") == 1).alias("meets_phone"),
            (pl.col("sedentary_hours_per_day") >= 6.0).alias("meets_sedentary"),
        ])
        .with_columns([
            (
                (pl.col("kidney_transplant_flag") == 1)
                | (pl.col("severe_cognitive_impairment_flag") == 1)
                | (pl.col("unstable_cad_flag") == 1)
                | (pl.col("orthostatic_hypotension_flag") == 1)
                | (pl.col("amputation_flag") == 1)
                | (pl.col("severe_psych_neuro_flag") == 1)
                | (pl.col("ortho_condition_flag") == 1)
                | (pl.col("mobility_device_dependence_flag").fill_null(0) == 1)
                | (pl.col("unable_walk_block_flag").fill_null(0) == 1)
                | (pl.col("unable_stand_without_assist_flag").fill_null(0) == 1)
                | (pl.col("patient_advisory_board_flag") == 1)
                | (pl.col("ckd_stage_final") == 5)
            ).alias("has_structured_exclusion")
        ])
        .with_columns([
            (
                pl.col("sedentary_hours_per_day").is_null()
                | pl.col("language").is_null()
                | pl.col("ckd_stage_final").is_null()
                | pl.col("mobility_device_dependence_flag").is_null()
                | pl.col("unable_walk_block_flag").is_null()
                | pl.col("unable_stand_without_assist_flag").is_null()
            ).alias("needs_manual_review")
        ])
        .with_columns([
            pl.when(
                pl.col("has_structured_exclusion")
            )
            .then(pl.lit("not_eligible"))
            .when(
                pl.col("meets_race")
                & pl.col("meets_age")
                & pl.col("meets_ckd")
                & pl.col("meets_language_proxy")
                & pl.col("meets_phone")
                & pl.col("meets_sedentary")
                & ~pl.col("needs_manual_review")
            )
            .then(pl.lit("likely_eligible"))
            .otherwise(pl.lit("manual_review_required"))
            .alias("screening_status")
        ])
        .with_columns([
            pl.col("patient_id")
              .map_elements(stable_study_id, return_dtype=pl.String)
              .alias("study_id")
        ])
    )

    # Return a PHI-minimized output
    return (
        working
        .select([
            "study_id",
            "screening_status",
            "age_years",
            "ckd_stage_final",
            "meets_race",
            "meets_age",
            "meets_ckd",
            "meets_language_proxy",
            "meets_phone",
            "meets_sedentary",
            "has_structured_exclusion",
            "needs_manual_review",
        ])
        .sort(["screening_status", "study_id"])
    )

9) src/trial_eligibility/main.py

Code 
from __future__ import annotations

from pathlib import Path

import polars as pl

from trial_eligibility.db import read_polars, read_sql_file
from trial_eligibility.logic import classify_candidates
from trial_eligibility.settings import settings


def render_query(template: str) -> str:
    return template.replace("@ROW_LIMIT", str(settings.row_limit))


def main() -> None:
    output_dir = Path(settings.output_dir)
    output_dir.mkdir(parents=True, exist_ok=True)

    raw_sql = read_sql_file("sql/candidate_extract.sql")
    query = render_query(raw_sql)

    source_df: pl.DataFrame = read_polars(query)
    result_df: pl.DataFrame = classify_candidates(source_df)

    result_path = output_dir / "trial_candidate_prescreen.csv"
    summary_path = output_dir / "trial_candidate_prescreen_summary.csv"

    result_df.write_csv(result_path)

    (
        result_df
        .group_by("screening_status")
        .agg(pl.len().alias("patient_count"))
        .sort("screening_status")
        .write_csv(summary_path)
    )

    print(f"Wrote: {result_path}")
    print(f"Wrote: {summary_path}")


if __name__ == "__main__":
    main()

Key design decisions explained:

Missing data — SimpleImputer(strategy="median") handles all the missingness you described. Median is preferred over mean here because several lab values (AST, ALT, ALK, BUN) are heavily right-skewed.

Log transforms — Applied via Polars before the sklearn pipeline for AST, ALT, ALK, tbil, glucose, WBC, platelets, BUN, and creatinine. These are notorious for extreme outliers in clinical labs that can destabilize linear models and impair tree splits.

Class imbalance — class_weight="balanced" is set for both Logistic Regression and Random Forest, which automatically up-weights the minority class. GBM handles this implicitly through its loss function but you can add scale_pos_weight if imbalance is severe.

Three model types — Logistic Regression gives interpretable coefficients and a calibration baseline; Random Forest is robust to multicollinearity (which will be present among the CBC components like HGB/HCT/RBC); Gradient Boosting often wins on tabular clinical data.

Evaluation metrics — ROC-AUC, Average Precision (PR-AUC), and F1 are all reported because accuracy alone is misleading with class imbalance.

active feature — Included intentionally since active inflammation is clinically the most direct predictor of remission failure; if you want a “lab-only” model, simply remove it from FEATURES.

    if ismissing(x)

function iqr_bounds(v::Vector{Union{Missing, Float64}})
    x = collect(skipmissing(v))
    if length(x) < 4
        return (missing, missing)  # too few points
    end
    q1 = quantile(x, 0.25)
    q3 = quantile(x, 0.75)
    iqr = q3 - q1
    return (q1 - 1.5*iqr, q3 + 1.5*iqr)
